Termination Request Notes:

*These steps must be used for JIRA and PartnerJIRA

·         Open up Termination Request from email using the link provided in the email.  Enter username and password if needed.

·         Log into another instance of production jira.

·         Log into production partnerjira instance.

·         In both instances, enter Administration->Users to search for usernames within the system

·         Use “Full Name Contains”  to search by name, not by username

·         Search by Last name or by First name Last name using name in the termination request. 

·         If person is a user in JIRA, click on their username on the left.  If not a user, follow directions in what do to next section at the bottom of this document.

·         Click on “Edit Groups”

·         Click on each group user is in, and click on leave

·         Click on “Return to viewing user “Person’s name”

·         Next, click on “View Project Roles”

·         Click on “Edit Project Roles” and uncheck any roles the user may be in.  Then click Save

·         Click on “Return to viewing user “Person’s name”

·         Lastly, click on “Edit Details”

·         Uncheck the “Active” box

·         Click update

·         Click on “Users” in the left nav bar to return to User Search page

·         The Full Name of the person should have a line through their name, meaning they are not an inactive user.

·         Do steps in: What do to next” section below. 

What do to next?

·         JIRA will be listed near the bottom, and will say “Incomplete”

o   If user was not in JIRA, replace “incomplete” with “Not an active user”

o   If user was in JIRA, replace “incomplete” with “Removed from all groups, project roles, and set inactive”

 

·         PartnerJIRA will be listed under JIRA, and will say “Incomplete”

o   If user was not in JIRA, replace “incomplete” with “Not an active user”

o   If user was in JIRA, replace “incomplete” with “Removed from all groups, project roles, and set inactive”

 

·         Assign issues in bulk to respective assignee or assign issue one-by-one. 

o   Create a filter with the following fields populated.  You can save this filter to use for future references.

§  Project: Service Request Lite - Strategic Tools

§  Issue Type: Task

§  Status: Opened

§  Assignee: Your Name

§  Component/Batch: Termination Requests

o   Once the filter is created or you have accessed the filter, click on the cog wheel in top-right corner, and click on the link under Bulk Change:

o   Choose all the issues, and click Next

o   Choose to Edit the Issues, then click Next

o   Change the assignee to Ronal Storey

o   Make sure there will be email notifications for these changes by seeing if the check box at the bottom is checked.

o   Confirm the changes to change the assignee fields for all the issues.
