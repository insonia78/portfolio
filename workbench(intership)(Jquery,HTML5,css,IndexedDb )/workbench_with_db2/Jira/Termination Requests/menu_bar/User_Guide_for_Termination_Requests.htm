Section A: Checking a user in Production JIRA

 

1.    Login into the Production Instance of RIJIRA

o   https://rijira.gtk.gtech.com:8443/jira/secure/Dashboard.jspa

 

2.    Open up Termination Request from email using the link provided in the email.  Enter username and password if needed OR open up the termination request in JIRA using a filter to filter the requests assigned to you.  (Easiest way is to use the emails)

Figure 1: Example Termination Requests in Email

Figure 2: Filter to use to search for Termination Requests assigned to you

3.    Log into another instance of Production JIRA (RIJIRA).  Only do this if needed if you decided to use a filter instead of the emails.

 

4.    In the first instance of RIJIRA you opened in Step 1, enter Administration->Users to search for usernames within the system.  Enter your password (If needed).

 

5.    Use Full Name Contains  to search by name, not by username

 

Figure 3: How to search for a user

6.    Search by Last name or by First name Last name using name in the termination request.

o   Ex) Bell OR Stephen Bell

o   **Be careful of extra spaces before and/or after last name or first name.  This will affect the results of the query.

 

7.    If person is a user in JIRA, click on their username on the left.  If not a user, in the termination request for this user, next to JIRA, replace Incomplete with Not an active user.  Move on to Section B.

Figure 4: Location of username

8.    Click on Edit Groups

Figure 5: Bottom navigation bar for next steps

9.    Click on each group user is in, and click on leave

Figure 6: Edit Groups Screen

 

10. Click on Return to viewing user Persons name. (You may need to scroll up on the page)

 

11. Next, click on View Project Roles on the bottom navigation bar.

 

12. Click on Edit Project Roles (At the bottom of the page) and uncheck any roles the user may be in.  Then click Save.  If the user is in no roles, you can move on to the next step.

Figure 7: Project Roles Page

13. Click on Return to viewing user Persons name

 

14. Lastly, click on Edit Details

 

15. Uncheck the Active box

Figure 8: Edit Details Screen

 

16. Click update

 

17. Click on Users in the left nav bar to return to User Search page.  The Full Name of the person should have a line through their name, meaning they are not an inactive user.

 

18.  In the termination request for the user, next to JIRA, replace Incomplete with Removed from all groups, project roles, and set inactive.

Figure 9: Example Termination Request

 

19. Move on to Section B to check to see if the user is a user in Partner JIRA.

Section C: Bulk Edit Termination Requests to next Employee

 

1.    Once all termination requests have been complete, they need to be assigned to the next person so they can complete their section.  The easiest way to do this is to bulk edit all the issues at once.  

 

2.    Create a filter with the following fields populated.  You can save this filter to use for future references.

                                          i.    Project: Service Request Lite - Strategic Tools

                                        ii.    Issue Type: Task

                                       iii.    Status: Opened

                                       iv.    Assignee: Current User

                                        v.    More Criteria ->Component/Batch: Termination Requests

Figure 17: Termination Request Filter

3.    Once the filter is created or you have accessed the filter, click on the cog wheel in top-right corner, and click on the link under Bulk Change:

Figure 18: Bulk Change

4.    Choose all the issues, and click Next

Figure 19: All Issues to Bulk

5.    Choose to Edit the Issues, then click Next

6.    Change the assignee to Ronal Storey

Figure 20: Who to assign issues to

7.    Make sure there will be email notifications for these changes by seeing if the check box at the bottom is checked. (At the bottom of the page)

Figure 21: Email Notifications

8.    Confirm the changes to change the assignee fields for all the issues, and the click Confirm.

Figure 22: Confirm Changes

 

Appendices: Special Cases of Termination Requests

 

A)     The user is already inactive in JIRA or Partner JIRA

 

1.       Check to see if there was already a termination request for this user by changing the filter to search for the users name with termination in front of it (Termination users last name), changing the Status of the issue to Closed, and changing the Assignee to All.  For example:

2.       If there is already a request, write down the issue ID (For example, SRPT-1923), go to the current termination request for this user and click on More Actions, and click on Link.  If there is not a duplicate request, in the termination request for JIRA and/or Partner JIRA, replace Incomplete with Removed from all groups, project roles, and set inactive.

Figure 23: More Actions

3.       On the link screen, choose Duplicate of for This Issue, and then in the Issue search box, search for the duplicate issue using the issue key written down in step 2.  Click on that issue.   Then click on Link.  A link to the duplicate issue will be created in the current issue.

Figure 24: Link Screen

4.       Lastly, close the current termination by clicking on Close Issue.

Figure 25: Close Issue
